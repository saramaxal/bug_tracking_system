## Как запустить
1. Необходимо создать базу данных `cfaccerr` на базе драйвера Apache Derby Embedded и каталог для проекта. 
2. Склонировать git repository на машину (не в каталог проекта)
3. Отредактировать build/copy_project.bat
- Назначить переменной `COMPILER` путь до `cfencode.exe` в вашей системе
4. Создать символьную ссылку (`link-to-project`) на каталог для проекта, созданный в первом пункте в build.
Команда: `mklink /D build\link-to-project path\to\project`
5. Если каталог репозитория открыт `vscode`, то можно скомпирировать ctrl+shif+b иначе от местоположения корня репозитория (пункт 2) запустить команду build\copy_project.bat из cmd. После этого в каталоге проекта (пункт 1) должен появиться проект.

## Описание

Цель **Bug tracking system**: возможность авторизированному пользователю вести учет по ошибкам, произведенным в процессе разработки. 

Система должна поддерживать:
- авторизацию пользователя
- создание и редактирование пользователя
- создание и редактирование ошибки
- просмотр списка пользователей и списка ошибок

В данный момент реализовано:
- Добавление ошибки
- Редактирование ошибки
- Добавление комментариев (оно работает, сейчас нужно обновить страницу)
- Созданы все таблицы 
- Авторизация

Таким образом в разработке был затронут цикл разработки от создания структуры базы данных, до заполнения и редактирования данных таблиц с помощью пользовательского интерфейса.

## О разработке
Сам проект позволил мне получить интересный и ценный опыт. Мне понравился способ, который предоставляет ColdFusion для создания динамичный страничек. 

### Идеи, возможности, перспективы
Неавторизированные пользователи не имеют доступа к системе, но абсолютно все, кто имеют, могут плодить ошибки и других пользователей. Думаю добавить разграничение прав доступа среди авторизированных пользователей, дающее только ограниченному кругу (админам) создавать новых юзеров.


### Figma
Для возможности наглядно оценить задачу был сделан дизайн в `Figma`. Пришло осознание недостатков *flexового* подхода при проектировании таблиц. Осознание проблемы поможет мне в ее решении и допиливании стилей в будущем :)
